GSys is a floodway monitoring application implemented with smart and mobile water gauges as first class citizens in the Web of Things to cater for and alert the users regarding the various water-level scenarios present within a floodway. This application can be used by anyone who is interested in viewing the water-level for a floodway hoewver for integrating a sensor within the application or to get notified about the varying water-levels, users need to get registered.

1.Registration

The Sign-up button can be clicked to start the registration process in order to reach the form as shown in figure below.

  • All fields are mandatory
  • E-mail should be entered with "@"
  • Passsword should be atleast 6 characters long

2.Home

Once registered, users can visualize the sensors geographically under the HOME tab. The sensors have been displayed through markers with different colors suggesting different level of risks associated with water level.


  • Passable (Green): Water level in the regions with this type of sensors is low and safe to pass-by
  • Risk(Orange): Water level in the regions with this type of sensors is moderately high and could be a risk to pass-by
  • High Risk(Red): Water level in the regions with this type of sensors is very high and should not be crossed
  • Emergency Station(Blue): Indicating near-by emergency stations
  • Service Station(Cyan): Indicating near-by service station

3.Sensors

A list of sensors deployed and registered within the application by different users can be visualized under the SENSOR tab as shown in the figure below. The sensor can also be visualized by clicking on the SHOW ON MAP button.

Sensors with green icon are the ones that are currently transferring data while the red ones are offline. In addition to this it also gives information about the sensor installation site i.e. river name and the crossing on which it is installed (bridgeor floodway).


4.Settings

Once the user has registered with the application, they can access the SETTINGS view through the drop-down menu to manipulate their account as shown in the figure below. The settings view provides options to:

  • Create new or adjust existing Sensors
  • Create new or adjust existing Thresholds
  • View the Subscribed Thresholds & Sensors

Sensors

The registered user can create new sensor by using the CREATE NEW SENSOR button under the sensor tab in settings as shown in the figure below


The Create New Sensor button opens up a form where user has to enter mandatory information related to sensor which are:

  • Description & Unique ID of Sensor
  • Privacy Settings for Sensor
  • Latitude, Longitude of Sensor
  • Selection of Waterbody on which sensor is to be installed
  • Crossing Type e.g. bridge and seasonal dependencies if the sensors reading are to be season specific
  • Sensor's height, Crossing height and sensor threshold
  • Measuring interval for normal & during a flood activity

Create New Sensor Form

After entering all necessary information and submitting it, user can then view the sensor under the sensors tab and can also view it on map by clicking on SHOW ON MAP button

Sensor Details

The user can also view additional details related to a particular sensor by clicking on that sensor under the sensor tab which include:

  • Real & Time-Series Data of Sensor
  • Nearby Sensors
  • Nearby Emergency Stations
  • Sensor Configurations

The Sensor Details provide three tabs General, Real-Time, Time-Series

General

Under the General Tab, User can view all the configurational information related to a sensor which has been entered while creating it. It also displays the location of sensor on map.

It also displays the last measured water-level,time associated with it and descriptive statistis related to the overall data.Users can subscribe to a sensor by clicking on the SUBSCRIBE button on top right corner.

The General Tab also provides weather updates around the sensor for hourly and daily basis. It also provides information regarding the other sensors which are in proximity to the sensor.The user can visualize all the sensor details related to these sensors too from configuration to incoming data by simply clinking on the name.


In addition to information about nearby sensors, information about nearby emrgency stations and service stations can also be viewed under the general tab and can be visualized on map by clicking on the map buttons provided next to their names.

Time-Series

The time-series tab displays the time-series data for a particular sensor. The user can adjust the duration for which they wish to visualize the time-series on the graph through the tabular options provided below the graph. The values for nay particular time on grpah can be read by simply hovering the cursor over to that point.

The graph also shows the measurement values and defined thresholds through which water-level is being measured and users are alerted about the varying water level scenarios

  • Gauge Zero: A fixed value that has been defined for the zero-level of sensor with reference to the water bed of waterbody
  • Sensor Height: Height of Sensor that has been defined while creating a new sensor within the application
  • Sensor Threshold Height: The threshold height defined while creating the sensor which is the maximum safe value for passing a crossing
  • Crossing Height: The height of the crossing or installation site where sensor has been placed
  • Warning threshold: The threshold value at which a warning is sent to the user to stay alert
  • Critical threshold: The threshold at which a danger alert is sent to the user and suggest not to pass the crossing site
  • Water Level: The level of Water at a particular time

Real-Time

The Real time tab provides continuous data that is coming from sensor in real time without any post processing.

As the data is being visualized simulatneously as being fetched so it can be compared with the sensor height and crossing height to generate warnings. The statistics at the top are generated on basis of all the data collected by the sensor. The thresholds are not being compared in real time data because no cumulative calculations have been performed.

Thresholds

User can also define new threshold for recieving alerts with respect to their vehicles by clicking on the CREATE NEW THRESHOLD button under the threshold tab within settings.

  • Type: Selection from a list of Vehicles stored in database
  • Description: Brief Description about thresholds
  • Warning & Critical Threhold: Alert & Danger level heights depending on the type of vehicle

All thresholds defined by a single user can be viewed under the Thresholds tab after being logged in. The thresholds can also be edited through the EDIT button provided adjacent to each threshold.


Subscription

The subscription tab allows the user to view all the subscribed or installed sensors by that particular user and the thresholds that have been defined by the user and to delete if any.
